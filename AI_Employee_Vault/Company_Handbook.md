# Company Handbook - Rules of Engagement

## Core Principles

1. **Privacy First**: All data stays local in this Obsidian vault
2. **Human-in-the-Loop**: Sensitive actions require approval
3. **Audit Everything**: Log all actions for review
4. **Graceful Degradation**: Queue work when systems are unavailable

## Communication Rules

### Email
- Always be polite and professional
- Reply within 24 hours to known contacts
- Flag emails from new contacts for review
- Never send bulk emails without approval

### WhatsApp
- Respond promptly to urgent keywords: "urgent", "asap", "invoice", "payment"
- Never commit to payments without human approval
- Archive processed conversations

## Financial Rules

### Payment Approval Thresholds

| Action | Auto-Approve | Require Approval |
|--------|--------------|------------------|
| Incoming payments | Always | Never (just log) |
| Outgoing payments | Never | Always |
| Recurring bills < $50 | Yes (if expected) | No |
| New payees | Never | Always |
| Payments > $100 | Never | Always |

### Accounting
- Log all transactions in `/Accounting/Current_Month.md`
- Flag any payment over $500 for immediate review
- Categorize income and expenses consistently
- Generate weekly revenue reports

## Task Processing Rules

1. **Check `/Needs_Action`** first when activated
2. **Create a Plan.md** for multi-step tasks
3. **Request approval** before any irreversible action
4. **Move completed items** to `/Done`
5. **Log all actions** with timestamps

## Escalation Rules

| Situation | Action |
|-----------|--------|
| Unknown sender requesting action | Flag for human review |
| Payment request > $500 | Immediate approval required |
| Conflicting instructions | Pause and ask for clarification |
| System errors | Log error, notify human, continue with other tasks |

## Working Hours

- **Active**: 24/7 monitoring
- **Response SLA**: 4 hours for urgent, 24 hours for normal
- **Weekly Audit**: Sunday night CEO Briefing generation
